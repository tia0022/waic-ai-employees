## 设计原则

### 步骤经济性（Step Economy）

> "每个小明尽量要控制好步骤的总数量，保证成本相对可控和效果的平衡。"

- 单个小明每次任务 **≤ 3 步**
- 全场景总步骤 **≤ 15 步**
- HITL 节点 **≤ 4 个**
- 每步必须回答：做了什么 + 省了什么 + 留了什么证据

### 7 维度框架

1. **岗位职责**（Job Definition）
2. **Skill**（能力原子）
3. **工作流**（3 步精简版 + HITL）
4. **工具权限**（R/W/X + 真实产品 API）
5. **KPI**（可量化 + 可取证）
6. **审计链路**（9 点全覆盖 + 区块链存证）
7. **业务交付物**（格式 + 频率 + 受众 + 价值承诺）

### 权限模型

- **R**（Read）：读取数据，自动执行
- **W**（Write）：写入内部系统，自动执行
- **X**（Execute）：对外提交/下单，**必须 HITL 前置签字**
