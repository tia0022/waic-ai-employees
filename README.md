## 七维结构

每位"小明"都按统一七维展开：

1. **岗位职责（Job Definition）** — 含"不做什么"边界
2. **Skill 清单（Atomic Skills）** — 原子 Skill + 职能映射 + 跨员工边界
3. **工作流（Execution Workflow）** — 含 HITL 节点与异常路径
4. **工具权限（Tool Permissions）** — R/W/X 三级；X 权限必须 1:1 匹配 HITL
5. **KPI** — 时效 / 准确率 / 成本 / 业务结果四维
6. **审计链路（Audit Trail）** — 数据采集 / 决策记录 / 人工裁决三层
7. **业务交付物（Business Deliverables）** — 看板 / 证据包 / 数据接口三类

## 治理原则

> **可读、可判、可建议；高风险动作必须可确认**

所有涉及对外提交、资金动作、批量发送、合同承诺的 X 权限调用，前置必须有 HITL 决策卡。
