## 一、本次修改核心内容

### 1. 派单屏布局重构（解决拥挤问题）

**问题**：主控派单屏（Scene 1）原本使用三行堆叠布局，组织架构图、依赖流程图、定时任务行各占一行，导致内容严重压缩，文字和节点互相遮挡。

**方案**：改为两行布局

```
旧：[组织架构图 34%]
    [依赖流程图 42%]
    [定时任务    1fr]

新：[组织架构图 50% | 依赖流程图 50%]  ← 横向并排，各占一半宽度
    [定时任务 全宽]
```

**技术实现**：
- `.dispatch-grid` 的 `grid-template-rows` 从 `34% 42% 1fr` 改为 `58% 1fr`
- 新增 `.dispatch-top` 两列网格容器（`grid-template-columns: 1fr 1fr`），包裹前两块 panel
- 保留 `.dispatch-grid > .panel` 样式作用于第三行任务面板

---

### 2. 全局 Agent 点击跳详情

**问题**：演示中所有 Agent 卡片仅用于展示，无法点击查看具体工作内容，缺乏交互深度。

**方案**：所有 Agent 可视化元素均绑定 `onclick`，点击即跳对应详情。

**新增统一分发函数 `openAgent(id)`**：

```javascript
function openAgent(id) {
  if (WORK[id]) openAgentModal(id);   // 有完整工作台 → 全屏 workbench
  else openAgentMini(id);              // 仅基础信息 → 弹出 mini 卡片
}
```

**覆盖范围**：

| 元素类型 | 所在位置 | 点击行为 |
|---|---|---|
| `org-node`（10个） | 派单屏组织架构图 | 核心5个→完整工作台；支持5个→mini卡 |
| `dep-node`（10个） | 派单屏依赖流程图 | 同上 |
| `agent-mini`（5个） | 顶部全局状态栏 | mini 卡片 |
| `matrix-card`（16个） | 收尾总览矩阵 | 核心Agent→工作台；扩展Agent→mini卡 |
| `archive-card` | 底部归档区 | 全屏 workbench（原有，保留） |

### 3. 交互反馈优化

- `dep-node` 悬停时 `border-color` 变为 Agent 主色、背景微亮，提示可点击
- `org-node` 悬停追加 `box-shadow` glow 效果（使用 `color-mix` 自适应主色）

- 所有可点击元素统一添加 `cursor: pointer` CSS 规则
