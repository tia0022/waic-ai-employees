# WAIC Demo · 白话版

> 给老板、销售、设计、运营同事看的。10 分钟读完。
> 完整技术 PRD 见 v4 文档。

---

## 一句话讲清这个 Demo 是干嘛的

我们要在 WAIC 大屏上,用 **5 分钟**、**一只景德镇茶杯卖到洛杉矶**的故事,让路过的观众一眼看出来:

> 这不是一个 AI 聊天工具。这是一个跨境 AI 员工平台,**后面有 16 个 AI 小明在协同工作**,能把出海这件复杂事自动跑通。

---

## 跟当前版本(v2)比,差在哪儿

打个比方:

- **v2 现在长得像微信聊天** — 一问一答、从上到下滚、密密麻麻全是字
- **v4 要长得像 NASA 任务控制室** — 头上一个地球转着,中间是当前哪个员工在干活,底下是已经干完的员工

这是从"客服式"升级成"指挥中心式"的根本差别。

## 观众的 5 分钟会经历什么

### 0:00 — 0:30 · 开场:全屏地球 + 一句话目标

观众一靠近展位,看到一个**全屏的 3D 地球**慢慢转着,景德镇、深圳港、洛杉矶三个点在脉动发光。

地球下方浮着一张大卡片,上面是一句话:

> 我是 **Amazon 美国站卖家**,要把 **景德镇茶杯 ×500 只** 卖到 **洛杉矶**,通过 **Amazon + TikTok Shop**,目标 **30 天** 内完成首批交付与正向利润核算。

加粗的 5 个词都是可以点击切换的(填空题),换一个,地球上的标点跟着动。

观众点大按钮"下达指令" → 地球用 0.8 秒收缩到屏幕顶部,变成一个 200px 的小窗口,主舞台开始干活。

**这一屏要传达:跨境 = 中国到世界 = 必须用地球讲,不能用文字讲。**

---

### 0:30 — 1:00 · 派单:让观众看见"平台的大脑"

主舞台变成一张组织架构图。

中间是 **"主控编排小明"** — 平台大脑。它下面分三层排列:

- **第一排:5 个核心小明**(经营评估 / 达人营销 / 合规 / 物流 / 分析师)
- **第二排:5 个支持小明**(支付结汇 / 客服售后 / 白盒存证 / 风控反欺诈 / 库存补货)
- **第三排:5 个扩展小明**(选品发现 / 知识产权 / 数据安全 / 品牌叙事 / 价格情报)— 灰色,本次不上场

旁边还有一张流程图,显示这次任务**有哪些小明同时开工、哪些是被触发的**。

下方一行字:**"主控已为你创建 7 项定时任务"**,比如:

> ⏰ T+7 天 · 主动扫描港口罢工情报
> ⏰ T+15 天 · 监控达人样品到达,启动差评通道

这一屏的目的:**让观众相信这是一个能主动调度、能定时执行、能并行干活的平台,而不是一个被动应答的对话机器人。**

---

### 1:00 — 4:00 · 干活:5 个核心小明依次上台

每个小明一张**全屏工作台**,5 个轮流播放。

每屏的模板都一样:

```
头像(有人脸,不再是字母方块)
名字 + 岗位 + T+几小时 + 量化 KPI

──────────────────────────

左边大字:业务结果
"我已为你找到 17 位洛杉矶本地高匹配达人"
"预计触达 50 万 + 潜在买家"
"首批 3 位达人已确认接受样品"

右边小卡:实时输出物
📱 达人匹配报告
🛍 Listing 草稿
🎬 TikTok 脚本

──────────────────────────

▼ 想看技术细节的点这里展开

底部:[需要确认] 锁定资产库  [授权 ▶]
```

**核心改变:** v2 的中间区有十几条技术日志("正在扫描达人数据库,筛选条件...")— v4 全部砍掉,只留三句业务大白话。技术细节折叠在抽屉里,想看的点开,不想看的看不见。

---

#### 重点:每个核心小明完成时,会触发一个"配角闪现"

5 个支持小明不占全屏,通过**右下角 2.5 秒的小气泡**露脸。

| 哪个核心干完 | 触发哪个配角闪现 | 弹出什么 |
|---|---|---|
| 经营评估小明完成 | 💱 支付结汇小明 | "已锁定汇率 1USD=¥7.18 · SAFE 申报包就绪" |
| 达人营销小明完成 | 📞 客服售后小明 | "差评监控通道已启动 · EN/ES 多语言应答" |
| 物流小明切换空运后 | 🛡 风控反欺诈小明 | "Amazon 账户健康分 98 · 无异常订单" |
| 分析师小明算完利润 | 📦 库存补货小明 | "建议加购 200 只到 LA 仓"(直接出现在行动清单里) |
| 全部完成时 | 🔐 白盒存证小明 | 作为收尾屏主持人正式登场 |

**为什么用闪现而不是全屏:** 闪现 = 让观众知道这个小明存在 + 看到它的核心输出,但不打断主流程。如果每个都给全屏,5 分钟肯定讲不完。

---

#### 已完成的小明会收纳到底部

每个小明干完,会**缩小成一张小卡片飞到屏幕最底部的"归档层"**,像收纳柜一样存好。

观众想看哪个干完的小明,点一下小卡片就能再次展开看详情。

### 4:00 — 4:45 · 结算:利润瀑布 + Compliance Vault

主舞台变成经营仪表盘,分三块:

**左上 — 利润瀑布图**
售价 ¥490 → 一项一项扣减成本(出厂、头程、空运、尾程、广告、税、汇损)→ 最后净利润 ¥164 用大字弹出,旁边一个利润率 33.5% 的环形进度条。

**右上 — 今日行动清单**
4 条建议,**每条都标明是哪个小明给的**(小头像背书),让观众看到这不是 AI 凭空建议,是有具体岗位在背后负责。

**下方 — Compliance Vault 实时存证流**
9 份关键单据(报关单 / FDA 报告 / 银行流水 / 退税包 / AI 决策快照 ×5)依次"飞"进 Vault,每份打上唯一的哈希值。

**这里有一个 Demo 最大的杀手锏:**

> 屏幕中央一个按钮 **「回放任意决策」**,点下去随机展开一份 AI 决策快照,显示:
> - 当时输入了什么数据(哈希)
> - 调用的是哪个模型 / 哪个版本
> - 命中了哪些规则
> - 输出了什么
> - 谁在什么时间确认的

**这一秒是整个 Demo 的"白盒可信"卖点的临门一脚。** v2 这部分藏得很深,v4 必须做成显性大按钮,让观众一眼记住。

## 几个设计决策的"为什么"

**Q1:为什么是 16 个小明,不是 5 个或者 50 个?**
5 个 — 太薄,显得平台单薄、故事窄。
50 个 — 太密,5 分钟讲不完,观众记不住。
16 个分四层 — 既显得"宽"(平台广度),又不会冲掉主线(5 分钟焦点仍在 5 个核心上)。

**Q2:为什么开场要用全屏地球?**
跨境的本质 = "中国到世界",这件事用地球讲 30 秒,胜过用文字讲 30 句。视觉冲击力是 WAIC 现场最稀缺的资源。

**Q3:为什么每个 Agent 要全屏一页,不像 v2 那样并排?**
v2 的问题:聊天气泡密集 + 文字爆炸 + 三米外看不清。
全屏页的好处:大字大数字大头像,展位大屏 3 米外也能看清。

**Q4:为什么要有"展开技术工作流"折叠按钮?**
WAIC 现场会同时来两类观众:
- 销售总监 / 老板 → 只想看业务结果
- 技术专家 / 投资人 → 想看技术细节
两类人不能在同一屏上同时满足 — 默认显示业务视图,点开才是技术视图。

**Q5:为什么 5 个支持小明只闪现 2.5 秒?**
2.5 秒刚好够看清头像 + 读完一句话 + 留下印象,不会打断主流程。
要让观众"知道这些小明存在",但又不能让他们"读完所有支持小明的内容"。

**Q6:为什么要留 5 个扩展小明在 catalog 里?**
让观众感觉到"这个平台还有更多" — 这只茶杯只用了 11 个小明,你的业务可能要用更多。
这是给观众的**想象空间**,也是平台后续可销售的能力清单。

**Q7:为什么"回放任意决策"这个按钮必须显眼?**
"白盒可信"是这个平台最大的差异化卖点,但 v2 把它藏在了文字里,根本看不出价值。
v4 必须把这件事做成**可点击的交互**:点一下,弹出"这个 AI 决策是怎么做出来的全过程"。
这一秒值整个 5 分钟。

## 现场怎么操作

**演讲者用** — 一根遥控笔 / 一个键盘:
- 空格 = 推进下一步
- 数字键 1-7 = 直接跳到某个场景
- ←/→ = 在并行的 Agent 之间切换焦点

**观众用**(如果是触摸大屏):
- 点 5 个 token 切换演示场景
- 展开技术细节抽屉
- 点 Vault 单据看审计回放
- 点扩展 catalog 看更多能力

**展位常态** — 没人操作时,**静默 90 秒自动重置**回到全屏地球开场,迎接下一波观众。

## 实施怎么做

不是一口气写完整个 Demo,而是**分 8 步走**(M1 → M8),每一步交付一个能在浏览器打开的可运行版本:

| 步骤 | 做什么 | 时间 |
|---|---|---|
| M1 | 搭骨架(布局 + 配色 + 字体) | 1-2 天 |
| M2 | 启动屏 + 全屏地球 + 收缩转场 | 1.5 天 |
| M3 | 顶部指挥层小窗 + 时间轴 + 16 Agent 状态条 | 1 天 |
| M4 | 派单屏 + 16 Agent 组织架构图 + 流程图 | 1.5 天 |
| M5 | 5 个核心小明全屏工作台 + 配角 toast 闪现 | 2 天 |
| M6 | 利润仪表盘 + Vault + 回放决策按钮 | 1 天 |
| M7 | 收尾屏 + Before/After + 16 小明全家福 | 1 天 |
| M8 | 打磨 + 自动循环 + 兼容性测试 | 1 天 |

**总耗时大约 10 个工作日。**

**为什么要分步:**
- 每一步交付时你能看一眼,确认方向对了再继续
- 出问题立刻能改,不会等到全部做完才发现走错路
- AI 编码工具(Claude Code / Cursor)一次性做完容易出错,分步质量更稳
