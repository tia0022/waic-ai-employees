# WAIC Demo · 设计师视觉脑图

> **用法:** 给视觉/UI/动效设计师。不讲业务逻辑,只讲视觉决策。
> **配套:** 完整 PRD 见 v4 文档,但本文档可独立使用。

## 🎨 1. 整体氛围(Mood)

### 1.1 关键词

> **指挥中心 · 战略地图 · 太空感 · 暗夜 · 冷峻 · 科技但有温度**

### 1.2 反义词(避免长成这样)

> ❌ 微信聊天 / 浅色文档 / 通用 AI 产品 / 钉钉式深蓝 / 紫色渐变 SaaS

- ✅ **Bloomberg Terminal** — 数据密集但冷静

- **不要用 Agent 色做大面积填充**(会让屏幕显得幼稚)

## ✏️ 3. 字体系统

### 3.1 字体选择(必须用,别替换)

| 用途 | 字体 | 来源 |
|---|---|---|
| 显示字体(标题、大数字) | `Manrope` 或 `Sora` | Google Fonts CDN |
| 中文正文 | `Noto Sans SC` | Google Fonts CDN |
| 数字等宽 | `JetBrains Mono` | Google Fonts CDN |

### 3.2 禁用字体

> ❌ `Inter` / `Roboto` / `SF Pro` / `Arial` / `PingFang` 系统默认
>
> 原因:这些会立即让 Demo 看起来像"通用 AI SaaS 产品",失去辨识度。

### 3.3 字号阶梯

```
48px   利润数字、关键大数字            (Manrope 700)
36px   场景标题、阶段名                (Manrope 600)
24px   Agent 名字、卡片大标题          (Manrope 600)
18px   业务描述句、卡片正文            (Noto Sans SC 500)
16px   普通正文、按钮                  (Noto Sans SC 400)
14px   说明文字、标签                  (Noto Sans SC 400)
12px   辅助信息、技术日志              (JetBrains Mono 400)
11px   timestamp、hash 等密集信息      (JetBrains Mono 400)
10px   极小标签、状态徽章              (Noto Sans SC 600)
```

### 3.4 字距与行高

- **大数字(36px+):** 字距 -0.02em(更紧凑,有指挥中心感)
- **中文正文:** line-height 1.6
- **英文/数字密集块:** line-height 1.4

## 🌍 5. 关键场景视觉规范

### 5.1 启动屏 · 全屏 3D 地球

- **背景:** 深空星空(`#070a12` + 微噪点 + 远景星点)
- **地球:** 暗色海洋 `#0a1428`,大陆轮廓 `#3a7bd5` 1px 描边,无国家填充
- **节点:** 脉冲光圈,颜色 = Agent 主色,直径 8-12px,呼吸 1.2s
- **路线:** 流光曲线,空运高弧 / 海运低弧,3s 循环
- **目标卡片:** 毛玻璃 overlay,带 1px 微高光边
- **大按钮:** 品牌色 #6366f1 + 呼吸光晕 + 24px 圆角

### 5.2 派单屏 · 组织架构图

- **节点视觉层次**
  - L0:120×80px,边框 2px,发光最强
  - L1:100×70px,边框 1.5px
  - L2:80×60px,边框 1px dashed,亮度 70%
  - L3:60×50px,**灰度处理**,边框 1px dotted,hover 才彩色
- **连线**
  - L0→L1:实线 1.5px + 流光
  - L1→L2:虚线 1px
  - L1→L3:无连线(hover 才出现潜在连接)
- **节点形状:** 圆角矩形,radius 8px

### 5.3 Agent 工作台 · 业务大字区

- **大数字:** 48px Manrope 700,Agent 主色
- **业务陈述句:** 18px Noto Sans SC 500,#e2e8f0
- **数字与单位的对齐:** baseline 对齐,单位略小一号
- **数字滚动入场:** countUp.js 或自写,400ms

### 5.4 利润仪表盘 · 瀑布图

- **正向 bar:** 青绿 #00c9a7
- **负向 bar:** 板岩灰 #64748b(不要用红色,免得显得"亏")
- **总计 bar:** 品牌靛蓝 #6366f1,放大 + 发光
- **bar 入场:** scaleY 0→1,200ms/bar,stagger 100ms

### 5.5 Compliance Vault · 文件飞入

- **单据视觉:** 文件夹图标 48×48,带哈希字段
- **飞入轨迹:** 从屏幕边缘抛物线进入中央 Vault 容器
- **入库后:** 文件夹变绿 + ✓ + 哈希值定格
- **主哈希展示:** 等宽字体 14px,字符 8 + ... + 字符 8

### 5.6 收尾屏 · 16 Agent 全家福

- **网格:** 4×4 矩阵,16 个 Agent 头像 + 名字
- **已激活(11 个):** 彩色 + ✓ 徽章
- **未激活(5 个):** 灰度 + "可调用"小标签
- **入场:** staggered 200ms/个,从中心向外扩散

## 👤 6. Agent 形象设计(头像)

### 6.1 风格规范

- **整体风格:** **半写实矢量插画**,中性偏专业
- **不要使用:** anime / Q 版 / 3D 卡通 / 拟人化动物
- **实现工具:** [DiceBear](https://www.dicebear.com/) `personas` 或 `notionists` 风格库
- **每个 Agent 用固定 seed**,保证刷新后形象稳定

### 6.2 尺寸与裁切

| 用途 | 尺寸 | 形状 |
|---|---|---|
| 工作台主头像 | 120×120px | 圆形 |
| 状态条头像 | 32×32px | 圆形 |
| 归档卡头像 | 24×24px | 圆形 |
| 行动卡背书头像 | 20×20px | 圆形 |

**L0 主控**
- 主控编排小明 · 中性 · 极简银灰深空风 · 胸前 "M" 徽章

**L1 核心 5(3 男 2 女)**
- 经营评估小明 · 男 · 深色西装 + 浅衬衫 · 银行家
- 达人营销小明 · 女 · oversized 西装 + T 恤 + 耳机 · 创意总监
- 合规小明 · 男 · 衬衫 + 领带 + 眼镜 · 律师/会计
- 物流小明 · 男 · 工装夹克 + Polo · 港口运营
- 分析师小明 · 女 · 极简衬衫 + 数据手表 · 投行分析师

**L2 支持 5(2 男 3 女)**
- 支付结汇小明 · 女 · 银行家正装 · 稳重

### 6.4 4 种状态变体

| 状态 | 表情 | 颜色 | 用途 |
|---|---|---|---|
| `idle` 待机 | 中性,眼睛朝下 | 灰度处理 | 未被激活 |

| `done` 完成 | 微笑 + 小手势(thumbs up) | 完整彩色 | 已交付 |
| `alert` 异常 | 皱眉 + 举手 | 红色边框 + 频闪 | 需人工确认 |

## ✨ 7. 动效规范

### 7.1 全局时长/缓动

| 类型 | 时长 | 缓动函数 |
|---|---|---|
| 微交互(按钮 hover) | 150ms | ease-out |
| 元素入场 | 400ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 场景转场 | 600ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 启动屏地球收缩 | 800ms | cubic-bezier(0.4, 0, 0.2, 1) |
| Toast 闪现 | 2500ms 总(入 200ms + 停 2000ms + 出 300ms) | ease-in-out |
| 循环动画 | 1.2-3s loop | linear / ease-in-out |

### 7.2 6 个必做的关键动效

#### ① 启动屏地球收缩(全场第一冲击点)

- 触发:点击"下达指令"按钮
- 时长:800ms
- 内容:地球 scale 1 → 0.18,position 中央 → 左上,同时目标卡片渐隐 → 化为顶部订单卡
- 缓动:cubic-bezier(0.4, 0, 0.2, 1)
- **务必抛光,这是 Demo 的"第一印象时刻"**

#### ② Agent 工作台转场

- 触发:进入下一个 Agent
- 时长:600ms
- 内容:上一个 Agent 向上飘起 + 渐隐(300ms)+ 头像缩小飞到底部归档层(400ms,可重叠)+ 下一个 Agent 从下向上滑入(300ms)
- 缓动:ease-out

#### ③ Toast 闪现

- 触发:核心 Agent 完成时,关联的支持 Agent 弹出
- 时长:2500ms 总
- 内容:从右下角滑入(200ms + 弹性)→ 停留 2000ms → 渐隐 + 上移 24px(300ms)
- 同时:顶部 Agent 状态条对应 Agent 从 idle → running → done

#### ④ 利润瀑布构建

- 触发:进入仪表盘
- 时长:总 3-4 秒
- 内容:每个 bar 从下向上 scaleY 0 → 1,数字滚动 0 → 实际值,stagger 100ms
- 最后总计 bar 多一个发光脉冲

#### ⑤ Vault 文件飞入

- 触发:进入存证屏
- 时长:每份 600ms,9 份 stagger 150ms,总约 1.8 秒
- 内容:文件图标从屏幕边缘抛物线飞入中央 Vault,飞入时旋转 30° + 缩放 0.6 → 1
- 入库后:绿色 ✓ + 哈希字段定格

#### ⑥ 16 Agent 全家福浮现

- 触发:进入收尾屏
- 时长:总 3.2s
- 内容:从中心向外辐射 staggered,200ms/个,激活的彩色 + ✓ 略大,未激活的灰度
- 完成后:可 hover 显示 KPI tooltip

### 7.3 循环动画

- **Agent 状态灯脉冲:** opacity 0.4 ↔ 1,1.2s loop,ease-in-out
- **地球路线流光:** 沿曲线移动,3s loop,linear
- **大按钮呼吸光晕:** box-shadow 强度变化,2s loop,ease-in-out
- **异常红色频闪:** opacity 0.3 ↔ 1,0.6s loop,ease-in-out
- **节点脉冲(地球上):** scale 1 ↔ 1.3 + opacity 0.6 ↔ 1,1.5s loop,ease-out

### 7.4 微交互

- **按钮 hover:** brightness 1.15,transform translateY(-1px),150ms
- **卡片 hover:** border 高亮 + 微小 lift(translateY(-2px)),200ms
- **可点击 Vault 文件:** cursor pointer + hover 时哈希字段下划线浮现

## 🚫 8. 反陈词滥调清单

### 8.1 绝对不要做

```
❌ 紫色 → 粉色渐变背景(每个 AI 产品都用)
❌ 旋转 spinner "AI 思考中..."(LOW 感)
❌ Inter / Roboto 字体(过于通用)
❌ Material Design 的浮起阴影(过时)
❌ 微信绿 / 钉钉蓝按钮(国产 SaaS 既视感)
❌ Emoji 当 logo(太随便)
❌ "AI 大脑"卡通形象(俗气)
❌ 闪光特效堆砌(降低专业感)
❌ 大量圆角(>16px,会显得卡通)
```

### 8.2 推荐做

```
✓ 严格深色(#070a12 起步)
✓ 大留白
✓ 单一显示字体(Manrope/Sora)+ 单一中文字体(Noto Sans SC)
✓ 等宽字体仅用于数字、哈希、时间戳
✓ 主色用于强调,深色用于沉浸
✓ 动画 < 1s 为主,关键转场可 800ms
✓ 圆角 6-12px(克制)
✓ 1px 描边 + 4-8px shadow(干净)
```

## 🎬 10. 第一眼测试(交付前自检)

设计稿出来后,请用以下 5 个问题快速过一遍:

1. **3 米外能看清主舞台的大数字吗?** — 不能 → 字号不够
2. **关掉颜色变成黑白,信息层次还清楚吗?** — 不清楚 → 字号/字重对比不够
3. **打开任意一屏,能立刻找到主视觉焦点吗?** — 找不到 → 信息密度过高
4. **像不像"另一个 AI SaaS 产品"?** — 像 → 字体/配色没破除陈词滥调
5. **如果只能展示一秒钟,观众能记住什么?** — 没法回答 → 缺少"signature visual"
