# WAIC 2026 跨境 AI 员工平台 Demo · v4 重构 PRD

> **本文档面向:** AI 编码助手(Claude Code / Cursor / Windsurf 等)
> **基线版本:** `waic-demo_0526_v2.html`(单页 3 栏 + 7 场景线性聊天)
> **重构目标:** 把"AI 客服式线性对话演示"升级为"经营指挥中心式多模态演示",强化交互、参与度、视觉冲击力
> **目标场地:** 世界人工智能大会 WAIC 2026 主舞台 / 展位大屏(假设 1920×1080,16:9)
> **目标时长:** 5 分钟主舞台脚本场 + 展位常态自动循环
> **v3 → v4 主要变更:**
> ① 启动屏全屏地球 → 进入主流程后缩为顶部 200px 小窗;
> ② 经营目标 token 从 9 个收敛到 5 个;
> ③ **新增 11 个 Agent**(L0 主控 1 + L2 支持 5 + L3 扩展 5),共 16 个 Agent,见 §5。

---

## 0. 设计原则(执行时请始终遵守)

1. **业务表达 > 技术表达**:正文区只放可被销售总监一眼读懂的商业结果;技术细节(API、字段、规则、模型版本)全部收进"展开"抽屉里。
2. **一次只看一个焦点**:同屏不允许出现两个以上需要观众"读"的密集文字块。
3. **并行工作必须被可视化**:多个 Agent 同时跑要在顶部状态条体现,不能藏起来。
4. **每个屏都要有"为什么这是 Agent 平台,不是单点工具"的视觉证据**:调度、依赖、回滚、定时、风控、协作必须看得见。**16 个 Agent 的存在感是"平台叙事"的核心证据**。
5. **拒绝"AI 通用紫色渐变 + Inter 字体"的陈词滥调**:保留 v2 的深色 + 多 Agent 多色系,但字体、动效、布局要重做。
6. **零外部 API 依赖**:所有数据写死在前端 JS 中,不调真实 LLM,不依赖网络。展会 Wi-Fi 不可信。
7. **核心 5 个 Agent 是主角,新增 11 个 Agent 是配角**:配角通过派单图露脸 + 主流程闪现 + 收尾汇总三种方式出场,绝不抢戏。

| 维度 | v2(现状) | v4(目标) |

### 2.2 一句话经营目标 · 5 token 收敛版

**模板句:**

> 我是 `[卖家身份]` ,要把 `[商品·数量]` 卖到 `[目的地]` ,通过 `[平台]` ,目标 `[天数]` 内完成首批交付与正向利润核算。

5 个 token 的可选项:

| Token | 默认值 | 其他选项 |
|---|---|---|
| 卖家身份 | **Amazon 美国站卖家** | TikTok Shop 卖家 / 独立站 DTC / 多平台卖家 |
| 商品·数量 | **景德镇手工茶杯 ×500 只** | 义乌玩具 ×2000 / 苏州丝巾 ×800 / 深圳 3C ×300 |
| 目的地 | **美国 · 洛杉矶** | 美国 · 纽约 / 德国 · 汉堡 / 英国 · 伦敦 |
| 平台 | **Amazon + TikTok Shop** | Shopify / SHEIN / 全平台 |
| 天数 | **30 天** | 15 天 / 60 天 |

**v3 中被砍掉的字段**(出发地/港口/运输方式)收敛理由:
- "出发地"已经在商品里隐含(景德镇茶杯 → 出发地=景德镇)
- "港口/运输方式"应该由 AI 推荐而非用户选择 — **这恰恰是物流小明的工作**,展示 AI 智能价值
- 这 3 个隐藏字段的结果会在派单屏由物流小明给出推荐

**交互:**
- 整句使用**打字机动效**逐字浮现,80ms/字符,~4 秒完成
- 每个 `[]` token 是 dropdown,点击展开候选项
- 改 token 时:**地球上的目的地标点动画切换**(联动),路线曲线弧高也变(空运高弧/海运低弧由后续 AI 推荐)
- 大按钮 **「下达指令 · 启动 AI 员工矩阵 ▶」**(品牌主色 + 呼吸光晕),点击触发地球收缩 + 进入派单屏
- 底部 3 个**快捷场景按钮**(让演讲者一秒切换演示主题),只需实现"景德镇茶杯·洛杉矶",其他 2 个 placeholder

### 2.3 启动屏 → 主流程的转场动画(关键)

点击 **「下达指令」** 后,**800ms** 内完成:

1. 一句话卡片向上飘起,渐隐(300ms)
2. 全屏地球收缩为顶部 200px 横幅(800ms,scale + position 联动)
3. 收缩过程中,目标卡片的内容**重组为顶部小窗的"当前任务条"**(显示在地球右上角)
4. 主舞台区(下方)同步显示派单屏内容(从下向上滑入)

这个转场是 Demo 的"第一个视觉冲击点",**实现质量决定观众的第一印象**,务必抛光。

### 3.4 右 35%:Agent 实时状态条(16 个 Agent 的展示规则)

- **L0 主控 + L1 核心 5 个 = 6 个 Agent 常驻显示**(头像 + 状态灯 + mini 进度,每个约 60×60px)

**状态规范(沿用 v3):**

| 状态 | 灯效 | 文字 |
|---|---|---|
| `idle` 待机 | 灰色静止 | 待机 |
| `queued` 排队 | 黄色弱光 | 排队 ·依赖 X |

| `done` 完成 | Agent 主色稳定亮 + ✓ | 已交付 |
| `alert` 异常 | 红色频闪 | 需确认 |

## 5. 完整 Agent 矩阵(共 16 个,新增 §)

> **v3 → v4 的最大新增**

### 5.1 分层与角色

| 层级 | 数量 | 在 Demo 中的呈现 | 目的 |
|---|---|---|---|
