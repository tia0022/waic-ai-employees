# WAIC 多 Agent 协同场景候选清单 V0.2

> **目的**：在 `协同场景候选清单_V0.1.md` 的基础上，把 WAIC 现场体验进一步压缩为 **3~5 分钟内可感知、可交互、可拿到结果** 的多 Agent 协同 Demo。
> **维护人**：合规/物流组 + 营销/经营评估/经营分析组共同 review
> **版本**：V0.2（2026-05-20）
> **输入来源**：`跨境AI员工平台_WAIC_v8.docx` 场景 A~E、WAIC 展位核心叙事、同期 GitHub 样板文件、AXON GLOBAL 网站、Score Hub 产品思路。
> **核心原则**：观众不是看 5 个聊天机器人，而是现场“下达经营目标 → AI 员工拆解派单 → 人在回路确认 → 拿到业务交付物”。

## 三、推荐组合（V0.2 建议）

### 方案一：3 分钟主体验（推荐）

| 序 | 场景 | 时长 | 角色 |
|---|---|---|---|
| 1 | **A 经营下滑根因诊断** | 3~4 min | 主 Demo · 经营分析主导 |
| 2 | **B 爆款达人资产锁定** | 3 min | 互动小 Demo · 营销主导 |
| 3 | **D DE 数字信用生成** | 3 min | 金融主题 Demo · 经营评估主导 |

**适配方式**：交互屏提供 3 个大按钮，观众任选一个；每个体验都在 5 分钟内拿到最终业务结果。中央大屏同步展示 Agent Mesh，观众离场前扫码领取对应报告。

### 方案二：讲解员主导组合

| 序 | 场景 | 时长 | 角色 |
|---|---|---|---|
| 1 | **C 供应链危机 + DE 周转资金** | 4~5 min | 主 Demo · 产融联动旗舰 |
| 2 | **B 爆款达人资产锁定** | 3 min | 辅助 Demo · 运营/营销观众 |
| 3 | **E 法国合规增长** | 2~3 min | 辅助 Demo · 合规/服务商观众 |

**适配方式**：适合销售或讲解员带队演示，能够讲清楚物流、合规、营销、金融的协同闭环。

### 方案三：极简循环播放

只演 **A 经营下滑根因诊断**，3~4 分钟一轮。每轮都收束为《CEO 今日行动清单》，最稳定，也最容易让路过观众理解“五位小明，即刻上岗”。

## 四、推荐场景详细脚本

### 场景 A：多平台经营下滑后的 CEO 经营复盘与组织派单

#### 1. 用户下达的终极业务目标

老板在交互屏选择：“最近生意变差了，请告诉我真正原因，并给我一份今天就能执行的行动方案。”

#### 2. 场景背景与突发危机

企业同时经营 Amazon 德国站、法国站和独立站。过去 14 天 GMV 下降 12%，但老板无法判断是广告变贵、库存不足、物流延迟、合规问题还是现金流压力导致。传统团队会各说各话：营销说预算不足，物流说 FBA 入库慢，财务说现金紧，合规说法国 EPR 有风险。现场要展示 AI 如何把部门噪音压缩成一份 CEO 可执行清单。

#### 3. 多 Agent 网格化协同链路（Agent Mesh）

| 时间轴 | 主导 Agent | 秒级执行流 |
|---|---|---|
| T+0 秒 | 经营分析小明 | 启动 COA 经营扫描，读取 GMV、毛利、广告、库存、物流、合规、现金流和达人数据。 |
| T+8 秒 | 经营分析小明 | 将经营下滑拆解为四类假设：流量问题、转化问题、库存问题、合规/资金问题。 |
| T+15 秒 | 经营分析小明 → 营销小明 | 要求分析广告 ACOS、关键词、达人内容断点与流量质量。 |
| T+20 秒 | 经营分析小明 → 物流小明 | 要求分析 FBA 入库延迟、库存可售天数和断货影响。 |
| T+25 秒 | 经营分析小明 → 合规小明 | 要求检查法国站 EPR/VAT 状态和高风险页面宣称。 |
| T+30 秒 | 经营分析小明 → 经营评估小明 | 要求评估现金流压力、资金缺口和融资必要性。 |
| T+60 秒 | 营销小明 | 回传：两个关键词 ACOS 高且转化下降，达人内容停更导致新客流量不足。 |
| T+70 秒 | 物流小明 | 回传：两个高利润 SKU 入库延迟，广告放量期间库存承接不足。 |
| T+80 秒 | 合规小明 | 回传：法国 EPR 资料未完全补齐，建议暂缓高风险推广话术。 |
| T+100 秒 | 经营评估小明 | 回传：21 天后出现现金流缺口，可用物流和退税证据准备短期授信。 |
| T+150 秒 | 经营分析小明 | 汇总根因：广告放量与库存承接错配 + 法国合规护栏不足 + 现金流提前量不足。 |
| T+180 秒 | 老板 HITL | 确认《CEO 今日行动清单》，系统派发任务并生成二维码报告。 |

#### 4. 协同中的核心数据流转图

| 数据发送方 | 数据接收方 | 传递数据 | 接收方处理动作 | 形成闭环 |
|---|---|---|---|---|
| 经营分析小明 | 营销小明 | GMV 下滑 SKU、广告账户、目标市场 | 分析 ACOS、关键词、达人内容断点 | 回传流量侧根因 |
| 经营分析小明 | 物流小明 | 受影响 SKU、销售趋势、库存数据 | 分析入库延迟与断货影响 | 回传供应链侧根因 |
| 经营分析小明 | 合规小明 | 法国站 SKU、页面内容、销售国家 | 检查 EPR/VAT 与平台政策风险 | 回传合规护栏建议 |
| 经营分析小明 | 经营评估小明 | 现金流、采购计划、退税预测 | 评估资金缺口与授信需求 | 回传融资建议 |
| 四位执行小明 | 经营分析小明 | 分析结果与可执行动作 | 汇总为 CEO 行动清单 | 形成统一经营结论 |

#### 5. 人在回路（HITL）交互点设计

| 决策卡 | 弹出时点 | 老板必须确认的内容 | 原因 |
|---|---|---|---|
| CEO 今日行动清单确认卡 | T+180 秒 | 是否接受广告降耗、库存补发、合规补齐、融资准备四类动作及优先级 | 涉及跨部门资源调度 |
| 预算与采购联动确认卡 | T+180 秒后 | 是否调整广告预算与采购节奏，避免再次出现库存承接错配 | 涉及经营策略改变 |

#### 6. 最终现场交付物包（Deliverables）

| 交付物 | 内容说明 | 审计级价值 |
|---|---|---|
| 《经营下滑根因诊断报告》 | 将 GMV 下滑拆解为广告、库存、合规、现金流四类原因，并给出证据 | 避免部门各说各话，形成统一经营真相 |
| 《CEO 今日行动清单》 | 列明 8 个可执行动作、负责人、时限、预期影响和审计来源 | 直接进入执行状态 |
| 《Agent 派单与闭环日志》 | 记录任务派发、回传结果、老板确认和后续跟踪 | 证明 AI 员工不是建议系统，而是协同执行系统 |
| 《7 日复盘看板》 | 跟踪 GMV、毛利、ACOS、库存、合规和资金状态变化 | 支持持续经营优化 |

### 场景 B：爆款达人突然出圈后的资产锁定与供应链联动

#### 1. 用户下达的终极业务目标

老板选择：“某条达人视频突然爆了，我希望马上放大销量，但不要出现样品混乱、重复邀约、库存断货和广告浪费。”

#### 2. 场景背景与突发危机

一个法国 TikTok 达人发布的视频在 6 小时内播放量突破阈值，评论区出现大量购买咨询。多个国家店铺准备同时联系该达人，存在内部撞单和重复报价风险。样品库存不足，德国站核心 SKU 库存偏低；如果无序放量，会导致样品混乱、广告浪费和断货反噬。

#### 3. 多 Agent 网格化协同链路（Agent Mesh）

| 时间轴 | 主导 Agent | 秒级执行流 |
|---|---|---|
| T+0 秒 | 营销小明 | 检测到达人视频播放、互动和评论增速异常，触发“爆款内容机会”。 |
| T+5 秒 | 营销小明 | 在中央达人资产库中锁定该达人，阻止多店铺重复建联。 |
| T+15 秒 | 营销小明 → 经营分析小明 | 同步视频表现、受众地域、评论关键词和预计转化区间。 |
| T+25 秒 | 经营分析小明 | 判断法国和德国站均有带动价值，但德国站库存风险较高，建议法国优先扩量。 |
| T+35 秒 | 营销小明 → 物流小明 | 发送样品补寄需求、预计订单增量和达人合作排期。 |
| T+55 秒 | 物流小明 | 回传库存约束，建议先补寄 30 件样品并将德国站广告切为库存保护模式。 |
| T+70 秒 | 营销小明 → 合规小明 | 提交达人二次合作 Brief、折扣话术和素材草稿。 |
| T+100 秒 | 合规小明 | 返回可用话术与禁用表达，提示不得使用绝对化功效表述。 |
| T+120 秒 | 营销小明 → 经营评估小明 | 同步样品成本、广告预算、订单预测，判断是否需要小额营销周转资金。 |
| T+150 秒 | 老板 HITL | 确认《爆款放大决策卡》，决定报价、样品规模、预算和国家优先级。 |

#### 4. 协同中的核心数据流转图

| 数据发送方 | 数据接收方 | 传递数据 | 接收方处理动作 | 形成闭环 |
|---|---|---|---|---|
| 营销小明 | 经营分析小明 | 达人视频数据、评论关键词、受众地域 | 判断利润空间与国家优先级 | 决定法国优先、德国限量承接 |
| 经营分析小明 | 物流小明 | 预计订单增量、库存风险、国家优先级 | 制定样品和补货安排 | 避免爆款带来断货 |
| 营销小明 | 合规小明 | 合作 Brief、折扣话术、素材草稿 | 输出合规可用话术 | 避免内容违规 |
| 营销小明 | 经营评估小明 | 样品成本、广告预算、订单预测 | 判断是否需要短期资金 | 支持增长资金安排 |

#### 5. 人在回路（HITL）交互点设计

|---|---|---|---|
| 爆款放大决策卡 | T+150 秒 | 确认达人报价、合作周期、样品数量、广告预算和国家优先级 | 涉及对外商务承诺与预算支出 |
| 达人资产长期合作确认卡 | 内容发布后 24 小时 | 是否将达人标记为长期合作资产并进入复购池 | 涉及品牌资产沉淀和后续商务策略 |

|---|---|---|
| 《爆款达人机会报告》 | 包含播放、互动、评论、受众和转化预测 | 支撑增长动作立项 |
| 《达人资产锁定记录》 | 记录达人归属、联系状态、报价、样品和合作历史 | 防止内部撞单和员工离职造成资产流失 |
| 《样品与库存联动计划》 | 明确样品寄送、补货节奏、库存上限和广告承载范围 | 防止爆款反噬供应链 |

| 《小额增长资金判断卡》 | 判断是否需要为样品和广告申请短期周转资金 | 将营销增长动作纳入资金计划 |

### 场景 D：备货资金缺口下的 DE 数字信用生成

#### 1. 用户下达的终极业务目标

老板选择：“我有真实订单和补货需求，但现金流暂时不够，帮我 3 分钟生成一份银行看得懂的经营评估材料。”

#### 2. 场景背景与突发危机

企业德国站核心 SKU 进入旺季，已有真实订单和稳定销售，但下一批补货需要提前支付采购款和空运费用。传统银行只看静态报表，看不清企业真实经营，因此企业很难快速获得低成本周转资金。经营评估小明需要把订单、物流、报关、退税、库存、广告和现金流证据整合为 DE 数字信用。

#### 3. 多 Agent 网格化协同链路（Agent Mesh）

| 时间轴 | 主导 Agent | 秒级执行流 |
|---|---|---|
| T+0 秒 | 经营评估小明 | 接收融资需求，弹出数据共享授权范围。 |
| T+15 秒 | 老板 HITL | 确认授权读取订单、物流、报关、退税、库存和财务摘要。 |
| T+30 秒 | 经营评估小明 → 物流小明 | 调取采购合同、运踪、补货 ETA 和运输真实性证据。 |
| T+45 秒 | 经营评估小明 → 合规小明 | 调取报关一致性、退税预测和税务风险等级。 |
| T+60 秒 | 经营评估小明 → 经营分析小明 | 调取现金流缺口、还款来源和利润预测。 |
| T+75 秒 | 经营评估小明 → 营销小明 | 调取爆款放量/广告预算用途和订单预测。 |
| T+110 秒 | 经营评估小明 | 运行 DE 评分和贸易真实性校验，生成建议授信额度。 |
| T+140 秒 | 经营评估小明 | 输出 2-3 套融资方案，显示额度、期限、利率和获批概率。 |

| T+180 秒 | 经营评估小明 | 生成《DE 数字化经营评估报告》和《银行授信材料包》。 |
